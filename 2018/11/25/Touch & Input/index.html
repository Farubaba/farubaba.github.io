Focus和Touch Mode以及requestFocusInTouchMode

传统不支持Touch Mode（触控操作）的设备上，用户操作基本都基于导航键(navigation-keys)或者轨迹球（trackball）来完成。因为用户界面上有很多UI,需要将命令发送到正确的UI控件上，就需要有Focus来确定当前应该由哪一个控件来接收用户输入。

在支持Touch Mode的设备上，可以根据用户事件发生位置，来确定哪个UI控件应该作为用户输入的接收者，所以基本上可以不需要Focus来辅助。当然，你也可以在Touch Mode模式下同时使用Focus，这样就需要在View中设置 isFocusableInTouchMode = true。

如果想要在Touch Mode模式下正确获取到焦点Focus，可以使用View.requestFocusInTouchMode()来代替 View.requestFocus();

On devices running Android 9 (API level 28) or higher, activities don’t assign an initial focus. Instead, you must explicitly request initial focus, if desired.

操作方式

名称 常见确认方式 描述
navigation-keys 【方向键】 “Enter” 键或者“OK”键 方向键，通常最少有上、下、左、右、确定、返回等按钮：使用实体方向键会使设备退出 Touch Mode
trackball【轨迹球】 press down the trackball（按下轨迹球） 轨迹球，类似翻转的机械鼠标，有个滚球突出来：使用轨迹球会使设备退出 Touch Mode
Touch【触控操作】 手指点击，滑动等 手指触摸屏幕，会使设备进入 Touch Mode

EventListeners

名称 类 触发场景 触发条件
onClick View.OnClickListener case 1、Touch（In Touch Mode）
case 2、Focus&Press “enter” key with navigation-keys or trackball
onLongClick View.OnLongClickListener case 1、Touch&hold（In Touch Mode）
case 2、Focus&Press&hold “enter” key with navigation-keys or trackball

 按住持续1秒
onFocusChange View.OnFocusChange This is called when the user navigates onto or away from the item, using the navigation-keys or trackball
onKey View.OnKeyListener 当焦点在View上时，用户press 、release 设备上的实体键时触发
onTouch View.OnTouchListener 当用户执行特定的 Touch Event 时触发，包括：一个press事件，一个release事件，或者还有多个move事件==>处理 MotionEvent 事件发生 within the bounds of the item
onCreateContextMenu View.OnCreateContextMenuListener

EventHandlers

名称 触发描述 其他
onKeyDown(int, KeyEvent) Called when a new key event occurs.
onKeyUp(int, KeyEvent) Called when a key up event occurs.
onTrackballEvent(MotionEvent) Called when a trackball motion event occurs.
onTouchEvent(MotionEvent) Called when a touch screen motion event occurs.
onFocusChanged(boolean, int, Rect) Called when the view gains or loses focus
