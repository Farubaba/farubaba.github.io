  2. Unicode编码和UTF-8编码

    Unicode一种编码，将世界上所有的符号都纳入其中。每一个符号都给予一个独一无二的编码，不存在编码转换，那么乱码问题就会消失。这就是 Unicode，就像它的名字都表示的，这是一种所有符号的编码，查看最新Unicode码表得知，目前Unicode字符集编码支持从0000~10FFFF 即从（0 ~ 1114111）一百多万个字符。所以Unicode字符二进制编码本身长度可以为 1 - 4 个字节，但Unicode只规定了符号的二进制代码，却没有规定这个二进制代码应该如何存储。

<pre>
[UTF-8、UTF-16、UTF-32]就是对Unicode编码的一种存储方案实现。具体实现方式见下表：
--------------------+---------------------------------------+-------------- 
Unicode符号范围      |        UTF-8编码方式                    | 存储字节数
(十六进制)           |         （二进制）                      |
--------------------+---------------------------------------+--------------
0000 0000-0000 007F | 0xxxxxxx                              |  1字节存储
0000 0080-0000 07FF | 110xxxxx 10xxxxxx                     |  2字节存储
0000 0800-0000 FFFF | 1110xxxx 10xxxxxx 10xxxxxx            |  3字节存储
0001 0000-0010 FFFF | 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx   |  4字节存储
</pre>

  1. Java基本数据类型存储和转换

    Java中所说的字符编码，简单理解就是字符char的二进制编码，Java字符（char）默认采用Unicode编码字符集，并默认使用UTF-8编码来存储这些Unicode字符（char）。不同版本Java，都只支持Unicode字符集的一个特定版本。Java 11将支持最新版的Unicode。Java 9实现了Unicode 8.0。JDK1.8中Character类型的BYTES = 2 ，并不支持2字节以上的Unicode字符。

    /**
 * 不同版本Java，都只支持Unicode字符集的一个特定版本。
 * 以下测试代码运行在JDK=1.8的环境中。
 * 
 * 以下Unicode编码字符并非文字，详细信息请根据编码值，查看Unicode码表。
 * https://www.unicode.org/Public/UCD/latest/charts/CodeCharts.pdf 
 * 
 * Unicode编码"262F"对应字符为：太极图案 ☯ ️
 * Unicode编码"262FF"对应文字为：上面一个横“目”，下面一个“曹”字（java8目前不支持，我的环境中无法显示，所以只能描述）
 * 
 * 输出结果为：
 * ☯ ️
 * ☯F -->并没有输出 \u262FF 对应的文字，只是输出了前4位（两个字节2626）的内容在Unicode码表中的对应符号☯，原样输出最后一个F️
 * 
 * 得出结论：Java8并不支持2字节以上的Unicode字符。{@link Character#BYTES} 为2，也正好说明了该事实。
 * 
 */
@Test
public void testJavaUnicodeSupport() {
    //Unicode = 262F表示太极符号：☯
    System.out.println("\u262F");
    //Unicode = 262FF表示上面一个横“目”，下面一个“曹”字.实际输出 ☯F
    System.out.println("\u262FF");
}

    |取值范围||0 ~ 65535|-128 ~ 127|-32768 ~ 32767|-2147483648 ~ 2147483647|-9223372036854775808 ~ 9223372036854775807|1.4E-45 ~ 3.4028235E38|4.9E-324 ~ 1.7976931348623157E308|

    byte(1字节) < char(2字节) = short(2字节) < int(4字节) = float(4字节) < long(8字节) < double(8字节)


    Byte =(-128 ~ 127)
    Character =(0 ~ 65535)
    Short =(-32768 ~ 32767)
    Integer =(-2147483648 ~ 2147483647)
    Float =(1.4E-45 ~ 3.4028235E38)
    Long =(-9223372036854775808 ~ 9223372036854775807)
    Double =(4.9E-324 ~ 1.7976931348623157E308)

    所以： 大于0的byte 和无符号 unsigned byte 和大于0的short 和无符号 unsigned short 都可以无失真地转化成对应的char，但int、long在大于char的最大值65535时，转化成char将失真。
